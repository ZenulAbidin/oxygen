Quick Start

Start the local stack, open the dashboard, create or select a merchant, then create a payment link or payment. Customers pay through the payment UI at the URL returned in the payment object. 

docker compose -p oxygen-local -f docker-compose.local.yml up -d --build

The dashboard runs at http://localhost:8080/dashboard. The local documentation runs at http://localhost:8081.

Demo credentials from docker.env.example:

email: your.address@gmail.com
password: your-password-at-least-8-chars

Create your first payment

  • Create an API token in Dashboard, Settings, Developer Section.
  • Configure webhook settings if your backend needs status notifications.
  • Use the Merchant API createPayment operation or create a payment link from the dashboard.
  • Redirect the customer to the returned payment URL.

Payment Flow

  1. The merchant creates a payment manually, through a payment link, or through the Merchant API.
  2. The customer opens the payment page and selects a supported payment method.
  3. OxygenPay monitors the selected address for an on-chain transaction.
  4. Mempool observations can be shown in the UI without changing internal payment state.
  5. After the required confirmations, the payment is marked successful and merchant balances are updated.
  6. Webhook notifications are sent when the payment changes state.

Creating a payment through the Merchant API

curl -X POST 'http://localhost:8080/api/merchant/v1/merchant/{merchantId}/payment' \
  -H 'X-O2PAY-TOKEN: {token}' \
  -H 'Content-Type: application/json' \
  --data '{
    "id": "8263313B-08C8-4855-8FAC-3A4CCBA7EAC2",
    "currency": "USD",
    "price": 30,
    "redirectUrl": "https://my-store.example/success",
    "description": "Sample Payment"
  }'

Payment IDs are provided by the merchant so repeated requests can remain idempotent. When a customer selects a currency, the payment method lock is time-limited; expired or underpaid payments fail.

Currencies and Fees

Payments can be priced in fiat currencies such as USD and EUR. Customers can pay with the supported crypto assets configured in the service.

Blockchain Assets Mainnet Testnet Minimum withdrawal
Bitcoin BTC mainnet testnet USD 40
Litecoin LTC mainnet testnet USD 40
Ethereum ETH, ETH_USDT, ETH_USDC 1 5 USD 40
Polygon MATIC, MATIC_USDT, MATIC_USDC 137 80001 USD 10

Incoming transaction service fees and withdrawal network fees are applied by the backend. Withdrawal fees depend on the blockchain and current network cost.

Status Meaning
inProgress A transaction was received but does not yet have the required confirmations.
success The transaction is confirmed and the merchant balance has been credited.
failed The payment expired, was underpaid, or otherwise failed validation.

Signed webhooks include an X-Signature header containing an HMAC signature over the raw request body. Each request also includes X-Webhook-Id and X-Webhook-Event headers for idempotency and routing.

Self-Hosting

The local compose file runs the all-in-one Oxygen process, Postgres, and this documentation server. The production pattern is the same: run Oxygen with Postgres, a persistent session/KMS path, public payment and webhook base URLs, and provider credentials or direct chain provider URLs as needed.

External provider keys are optional for local startup. BTC and LTC use Blockstream-compatible public explorer defaults unless overridden. EVM payment tracking needs JSON-RPC URLs unless Tatum is configured. TRON support needs a TronGrid API key for production use.

Common environment variables

DB_DATA_SOURCE=host=postgres sslmode=disable dbname=oxygen user=oxygen password=oxygen pool_max_conns=32
SESSION_FS_PATH=/src/tmp/sessions
SESSION_SECRET=session-secret
CORS_ALLOW_ORIGINS=http://localhost:8080
PROCESSING_WEBHOOK_BASE_PATH=http://localhost:8080
PROCESSING_PAYMENT_FRONTEND_BASE_PATH=http://localhost:8080
KMS_DB_DATA_SOURCE=/src/tmp/kms.db
CHAIN_BITCOIN_MAINNET_EXPLORER_URL=
CHAIN_BITCOIN_TESTNET_EXPLORER_URL=
